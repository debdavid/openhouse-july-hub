I have completed an Agent Brief for a Microsoft 365 declarative agent. Use it to write a complete system instruction ready to paste into M365 Copilot Agent Builder.

Apply the ARC, OEMGPC, and P-PASS frameworks as governing principles:

ARC (Aim, Route, Checkpoints): Establish the Aim of every request before producing output. Map the Route — the logical path to the output. Apply 2–3 Checkpoints to confirm the output meets the aim.

OEMGPC (Orient, Explain, Model, Guide, Pre-empt, Commit): Every output must orient the user, explain the core content, model it with an example where relevant, guide the user on what it means for them, pre-empt the most likely question, and commit with a clear next step.

P-PASS (Purpose, Persuasion, Audience, Safety, Standards): Before finalising output, check: does it serve the Purpose? Does it earn the Persuasion? Is the language right for this Audience? Are there Safety risks? Does it meet the professional Standards required?

Make the system instruction specific enough that this agent could not be mistaken for one built for a different professional in a different organisation.

What to do next

Three steps after you generate.

1

Quality check it

Paste the generated instruction back into the same Copilot chat and ask: "What is unclear or ambiguous in these instructions? What would you have to guess about rather than know?" Fix what comes up.

2

Save a copy

Copy the full instruction to a Word document or notes app. Name it: [Your Task] — System Instruction — July 2026. Save before you open Agent Builder.

3

Go to the Build Guide

Your instruction is ready. Go to Page 3 — the Build Guide — to configure your agent in M365 Copilot Agent Builder.

The rule: if your generated instruction is only a few sentences, your Agent Brief was too vague. Return to sections 2 and 3 of the brief, add more specific detail, then regenerate. A well-formed instruction is several hundred to several thousand words.
