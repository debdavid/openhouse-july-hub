01
Before you open Agent Builder

Generate your system instruction.

Copy the conversion prompt below. Paste your completed Agent Brief — all seven sections — into the space at the end. Send the whole thing to M365 Copilot in Work mode as one message. Copilot will generate your complete system instruction.

Conversion Prompt — paste into M365 Copilot (Work mode) 
I have completed an Agent Brief for a Microsoft 365 declarative agent. Use it to write a complete system instruction ready to paste into M365 Copilot Agent Builder.

The system instruction should:
- Define the agent's role and purpose precisely
- Specify the knowledge sources it draws from
- Describe the audience and communication style
- Set out the output format and structure in detail
- List what the human still decides and what the agent must never do
- Include a quality standard the agent applies to every output

Make it specific enough that this agent could not be mistaken for one built for a different professional in a different organisation.

Here is my completed Agent Brief:

[paste your completed Agent Brief — all seven sections — here]

If the generated instruction is only a few sentences — your Agent Brief was too vague. Return to sections 2 and 3, add more specific detail, and send the prompt again.

After generating

Three things to do before you open Agent Builder.

1

Quality check it

Paste the generated instruction back into the same Copilot chat and send this: "What is unclear or ambiguous in these instructions? What would you have to guess about rather than know for certain? List the gaps." Fix what comes up.

2

Save it to your own device

Copy the full instruction into a Word document or notes app. Name it: [Your Task] — System Instruction — July 2026. The sandbox closes when the session ends — your document does not.

3

Then go to Section 4 — the Build Guide

Your instruction is ready. Scroll down to Section 4 to configure your agent in M365 Copilot Agent Builder step by step.

Sandbox reminder: live connections to your email, Teams, and other channels may not be available today. Do not enter personal or sensitive data. Save your Agent Brief and system instruction before you leave. If you have your own Microsoft 365 subscription (Business or higher), you are welcome to use that account instead and carry your work forward from tonight.

04
Step-by-step · Have your system instruction ready

Build your agent in M365 Copilot Agent Builder.

Follow these steps in order. Your system instruction should already be saved before you start here.

1

Open M365 Copilot in Work mode

Go to microsoft365.com and sign in. Open Copilot. Look for the Work / Web toggle at the top of the Copilot panel. Select Work. If you see Personal mode, switch to Work — you cannot access Agent Builder in Personal mode.

2

Navigate to Agent Builder

In the left navigation panel, find the Agents section. Below any existing agents listed, click New agent. The Agent Builder screen opens.

3

Click Skip to configure

You will see a text input and below it a small link: Skip to configure →. Click the link — do not type in the box. This takes you directly to the configuration form where you paste your system instruction.

4

Name and describe your agent

Name: use the agent name from section 1 of your Agent Brief. Be specific — for example "Never Miss a Deadline" or "Progress Note Assistant". Not "My Agent."

Description: one sentence. What does this agent do and who does it serve?

5

Paste your system instruction

Find the Instructions field — the large text area. Paste your complete system instruction. All of it. A short instruction will not configure a useful agent.

6

Add knowledge sources

Permanent sources (upload here): templates, policy documents, brand guidelines, past examples. These stay connected to the agent permanently.

Per-request documents (do not upload here): files that change each time — today's meeting notes, a specific client brief. Attach these in the chat when making each request instead.

7

Add 3 starter prompts

Each starter prompt has two fields — Title (short label shown as a button) and Message (the full prompt sent when clicked). Make them specific to your three most common requests. Not "Write a report" — but "Draft a weekly status update using these meeting notes: [paste here]."

8

Switch to Preview and test

Click the Preview tab at the top of Agent Builder. Send a realistic test brief. Paste context directly into the chat if live data connections are not available. Does the output reflect your professional context without you explaining it?

9

Click Create

When satisfied with the output, click Create in the top right. Your agent is published and appears in the Agents list. If the output is not right — go back to Configure, update the Instructions field, and test again.

10

Save your system instruction before you leave

Make sure your system instruction is saved to a Word document or notes app on your own device. The sandbox closes when the session ends. Your instruction document does not.

You have built your agent. The Agent Brief and system instruction you produced today are professional documents that belong to you. Use them to configure your agent in your own M365 environment whenever you are ready.

05
Get better results every time

Tips & prompts.

Four things that improve every agent interaction — and a quality check to run before you use any agent output professionally.

Lead with the outcome, not the topic

Instead of: "Create a progress note."

Try: "Create an NDIS progress note for a session where [participant] worked on Goal 2. Observations: [notes]. Format: three paragraphs, plain English, no jargon."

The outcome tells the agent what the output needs to achieve. The topic alone does not.

Name the audience every time

Even though your system instruction includes audience context, naming the specific audience in each request sharpens the output.

Add: "Audience: family members of a resident — warm, plain English, no clinical terminology."

Give the most important thing first

For reports and summaries: state the key finding or decision upfront in your brief. The agent will treat it as the lead.

"The key finding is a 22% increase in response time. Build the status update narrative around what this means for the project timeline."

Use the quality check as a prompt

If the first output is not quite right, ask before you rewrite:

"Review this output. Does it reflect my professional context, or does it feel generic? Is there anything a professional in my field would find inaccurate? List what to fix, then rewrite."

Quality check

Five questions to ask before using any agent output professionally.

1

Does it serve the stated aim?

Ask: if I removed this section, would the audience miss anything essential? If not, cut it or merge it. Everything should earn its place.

2

Does the structure make sense?

Read it as a narrative. Does it build logically toward the conclusion? Would a sceptical reader find it convincing?

3

Is the language right for this audience?

Read one section aloud as if you are the person with the least familiarity with the subject. Is it still clear? Plain English wins over jargon every time.

4

Are there any risks in this content?

Check for unverified claims, content that could mislead, anything legally or reputationally sensitive. The agent does not know your legal obligations — you do.

5

Does it meet your professional standard?

Consistent format, appropriate length, correct terminology, alignment with regulatory or organisational requirements. The agent applies the standards you described — verify they were followed.
