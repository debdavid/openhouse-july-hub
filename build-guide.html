Step 3 of 3 · Configure your agent

End-to-end build guide.

Complete step-by-step instructions from Agent Brief to working agent. Follow each step in order. Your system instruction should already be saved before you start here.

Sandbox environment — today's M365 access is for this session only. You may not be able to test against live data. Focus on completing the configuration. The system instruction you save is yours to keep.

Phase 1 — Generate your system instruction

Before you open Agent Builder.

1

Open M365 Copilot in Work mode

Go to microsoft365.com and sign in. Open Copilot. Look for the Work / Web toggle at the top of the Copilot panel. Select Work. If you see Personal, switch to Work — you cannot access Agent Builder in Personal mode.

2

Paste the conversion prompt with your Agent Brief

Copy the full prompt from Page 1 of this hub. Paste your completed Agent Brief at the end — all six sections. Send it. Copilot will generate your system instruction. It should be several hundred words. If it is a few sentences, your Agent Brief was too vague — see the tip below.

3

Run the quality check

Paste the generated instruction back into the same Copilot chat and send this:

Quality check prompt
What is unclear or ambiguous in these instructions? What would you have to guess about rather than know for certain? List the gaps.

Fix what comes up. Update the instruction. Save the final version.

4

Save your system instruction

Copy the full instruction to a Word document or notes app. Name it: [Your Task] — System Instruction — July 2026. Do this before you open Agent Builder. This document is yours to keep.

Phase 2 — Configure in Agent Builder

Build your agent.

5

Navigate to Agent Builder

In M365 Copilot, look at the left navigation panel. Find the Agents section. Below any existing agents, click New agent. The Agent Builder opens showing "Build your own specialist agent."

6

Click Skip to configure

On the Agent Builder screen you will see a text input box and below it a link: Skip to configure →. Click the link. Do not type in the box. Skip to configure takes you directly to the configuration form where you paste your system instruction.

7

Name and describe your agent

Name: Use the agent name from Section 1 of your Agent Brief. Be specific — for example: Progress Note Assistant — Community Care or Weekly Status Update — WSP Infrastructure. A generic name like "My Agent" is hard to find later.

Description: One sentence. What does this agent do and who does it serve?

8

Paste your system instruction

Find the Instructions field — the large text area with placeholder text. Click inside it and paste your complete system instruction. Paste all of it. A short or summarised instruction will not configure a useful agent.

9

Add knowledge sources

In the Knowledge section, upload documents the agent will always draw on — templates, policy documents, past examples, brand guidelines. These are permanent sources.

Documents that change per request — today's meeting notes, a specific client brief — do not go here. Attach them in the chat when you make a request instead.

10

Add starter prompts

Add 3 starter prompts. Each has two parts:
Title: short label shown as a clickable button (e.g. "Draft a progress note")
Message: the full prompt sent when clicked (e.g. "Draft an NDIS progress note for a session with [participant]. Goals worked on: [list]. Observations: [notes].")

Make them specific to your most common requests — not generic placeholders.

Phase 3 — Test and publish

Test before you create.

11

Switch to Preview tab

At the top of Agent Builder click the Preview tab. This opens a test chat interface. Use it to evaluate your agent before publishing.

12

Send a real test brief

Send a realistic request — the kind you would actually send this agent in your working day. If you cannot access live data today, paste relevant context directly into the chat. Evaluate: does the output reflect your professional context without you explaining it? Does it match the format from your Agent Brief?

13

Refine if needed

If the output is not right: go back to Configure, identify the gap in the Instructions field, add the missing detail, return to Preview and test again. Common fixes: add more specific context, add explicit tone guidance, describe the exact output structure, add a non-negotiables section.

14

Click Create

When the agent is configured and tested, click Create in the top right. Your agent is now published and appears in the Agents list in the left navigation of M365 Copilot.

15

Save your system instruction

Before you close anything — make sure your system instruction is saved to a Word document or notes app. This is the real deliverable from today. The sandbox access ends with the session. Your instruction document does not.

You built your agent. The system instruction and Agent Brief you produced today are professional documents that belong to you. Use them whenever you have access to your own M365 environment.
