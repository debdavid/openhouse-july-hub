Tips, prompts & quality checks

Get better results every time.

Practical tips for improving your prompts, sector-specific starter examples, and a quality check to run before you use any agent output professionally.

Prompt tips

Four things that improve every agent interaction.

Lead with the outcome, not the topic

Instead of: "Create a progress note."

Try: "Create an NDIS progress note for a session where [participant] worked on Goal 2 (community participation). Observations: [notes]. Format: three paragraphs, plain English, no jargon."

The outcome tells the agent what the output needs to achieve. The topic alone does not.

Name the audience every time

Even though your system instruction includes audience context, naming the specific audience in each request sharpens the output.

Add: "Audience: family members of a resident in aged care — warm, plain English, no clinical terminology."

Give the agent the most important thing first

For reports and summaries: state the key finding or decision upfront in your brief. The agent will treat it as the lead.

Example: "The key finding is a 22% increase in response time. Build the status update narrative around what this means for the project timeline."

Use the quality check as a prompt

If the first output is not quite right, ask the agent to apply the P-PASS filter before rewriting:

"Before you rewrite this, check it against P-PASS: Purpose, Persuasion, Audience, Safety, Standards. Tell me what each check reveals, then rewrite."

Starter prompts by sector

Copy and adapt for your agent.

Health research
  • Summarise the following research papers for a non-specialist stakeholder audience. Flag the three most significant findings and any conflicts between papers: [paste abstracts or papers]
  • Draft a plain English grant application narrative section on [topic] using the following evidence: [paste data and citations]
Engineering & consulting
  • Draft a weekly project status update for [project name] using the following meeting notes and milestone tracker. Flag risks and blockers. Audience: [client name] — professional, concise: [paste notes]
  • Summarise the following site report into a two-paragraph executive summary for the client. Lead with progress, close with the one item requiring a decision: [paste report]
Disability & aged care
  • Draft an NDIS progress note for a session with [participant ID]. Goals worked on: [Goal 1, Goal 2]. Observations: [notes]. Format: three paragraphs, NDIS-aligned language, no jargon
  • Draft a family update for [resident reference] covering the week of [date]. Highlights: [list]. Tone: warm, person-centred, no clinical terminology
Real estate
  • Write a property listing for [address]. Bedrooms: [X]. Key features: [list]. Target buyer: [profile]. Brand tone: professional and aspirational. Maximum 120 words
  • Draft a vendor campaign update for week [X] of [Y] weeks. Enquiries: [X]. Inspections: [X]. Feedback themes: [notes]. Tone: transparent and confident
Self-employed & SME
  • Analyse my inbox from the past 24 hours and identify: (1) messages requiring a response today, (2) messages requiring a response this week, (3) messages that can wait. Draft a response for each priority-1 message in my professional voice: [paste email content]
  • Draft a client proposal for [client name]. Problem: [describe]. Proposed solution: [describe]. Investment: [amount]. Timeline: [dates]. Format: professional, lead with the client's problem
Quality check — P-PASS

Run this before using any agent output professionally.

Five questions. If any answer is no, the output needs work before you use it.

P

Purpose — does every part serve the stated aim?

Read each section. Ask: if I removed this, would the audience miss anything essential? If not, it does not have a purpose and should be removed or merged.

P

Persuasion — does the structure earn the argument?

Read it as a narrative. Does it build logically toward the conclusion or recommendation? Would a sceptical reader find it convincing? If the ending feels unsupported, the middle needs more evidence.

A

Audience — is the language right for this specific audience?

Read one section aloud as if you are the person with the least familiarity with the subject. Is it still clear? If not, it needs plain English revision.

S

Safety — are there any risks in this content?

Check for: unverified claims, content that could mislead, anything legally or reputationally sensitive, and anything that could be taken out of context. The agent does not know your legal obligations — you do.

S

Standards — does it meet your professional standard?

Check: consistent format, appropriate length, correct terminology, alignment with any regulatory or organisational requirements. The agent applies the standards you described — verify they have been followed.
